Elm Functions

Elm,Frontend development,Functions,Functional Programming<p>This article spotlights the central construct of the Elm programming language: Functions. What does the definition of a function look like, how can functions be chained via piping and what the heck is currying? Let us have a closer look …</p> <div id="more"></div> <h3>REPL</h3> <p>The <em>read-evaluate-print-loop</em> is a runtime environment for executing Elm code. The result of an expression is immediately shown, which makes the REPL an ideal tool for running short code samples and getting used to the syntax and language constructs. The REPL is part of the Elm standard installation and it can be opened on the command line:</p> <pre><code class="language-bash">$ elm-repl </code></pre> <p>The examples in this article can be run in the REPL. Multiline statements have to end with <code>\</code>. This is not the case for normal Elm code, but the examples in this article use the backslash so that you can copy, paste and execute them in the REPL. I’ve added the output of each expression in the examples as <code>-- comment</code>.</p> <h3>Calling functions</h3> <p>Before we take a look at how functions are defined, we will use functions from the <a href="http://package.elm-lang.org/packages/elm-lang/core/3.0.0/String">String module</a> which is part of the standard library. To use its functions we first need to import it.</p> <pre><code class="language-elm"><span class="hljs-keyword">import</span> String <span class="hljs-type">String</span>.append <span class="hljs-comment">-- &lt;function: append&gt; : String -&gt; String -&gt; String</span> <span class="hljs-type">String</span>.append <span class="hljs-string">"h"</span> <span class="hljs-string">"i"</span> <span class="hljs-comment">-- "hi" : String</span> </code></pre> <p>Calling a function without any argument gives us its signature. As you can see with <code>String.append</code>, the signature consists of the name and the types of the arguments and the return value. The type of the return value is not separated differently than the argument types, it is just the last element of the list.</p> <pre><code>&lt;function: append&gt; : String -&gt; String -&gt; String | function name | type argument 1 | type argument 2 | type result | </code></pre> <p>In Elm the <strong>arguments are separated with spaces</strong>, as you can see with <code>String.append &quot;h&quot; &quot;i&quot;</code>. There are also <strong>no parentheses around the arguments</strong>. Instead one uses parentheses to encapsulate the whole function call in case the result should be used as an argument for another function call. Which brings us to the topic of …</p> <h3>Piping: Chaining functions</h3> <p>Of course return values can be used as arguments for subsequent function calls. In Elm there are two ways to chain the calls:</p> <pre><code class="language-elm"><span class="hljs-type">String</span>.repeat <span class="hljs-number">3</span> (<span class="hljs-type">String</span>.toUpper (<span class="hljs-type">String</span>.append <span class="hljs-string">"h"</span> <span class="hljs-string">"i"</span>)) <span class="hljs-comment">-- "HIHIHI" : String</span> <span class="hljs-type">String</span>.append <span class="hljs-string">"h"</span> <span class="hljs-string">"i"</span> |&gt; <span class="hljs-type">String</span>.toUpper |&gt; <span class="hljs-type">String</span>.repeat <span class="hljs-number">3</span> <span class="hljs-comment">-- "HIHIHI" : String</span> <span class="hljs-type">String</span>.repeat <span class="hljs-number">3</span> &lt;| <span class="hljs-type">String</span>.toUpper &lt;| <span class="hljs-type">String</span>.append <span class="hljs-string">"h"</span> <span class="hljs-string">"i"</span> <span class="hljs-comment">-- "HIHIHI" : String</span> </code></pre> <p>Using the pipe-operators <code>|&gt;</code> or <code>&lt;|</code> one can pass through the result to the next function where it is used as the last argument for the function it gets piped into. The data can flow in both directions, though the usual way of chaining functions with the pipe operator is using <code>|&gt;</code> and having a single line per function call:</p> <pre><code class="language-elm"><span class="hljs-type">String</span>.append <span class="hljs-string">"h"</span> <span class="hljs-string">"i"</span> \ |&gt; <span class="hljs-type">String</span>.toUpper \ |&gt; <span class="hljs-type">String</span>.repeat <span class="hljs-number">3</span> </code></pre> <h3>Function definition and type annotation</h3> <p>Custom functions are defined with the name and arguments, which are separated from the function body by an equal sign. They can be written on a single line or using multiple lines:</p> <pre><code class="language-elm"><span class="hljs-title">sayHello</span> name = <span class="hljs-type">String</span>.append <span class="hljs-string">"Hello "</span> name <span class="hljs-comment">-- &lt;function&gt; : String -&gt; String</span> <span class="hljs-title">sayHello</span> name = \ <span class="hljs-type">String</span>.append <span class="hljs-string">"Hello "</span> name <span class="hljs-comment">-- &lt;function&gt; : String -&gt; String</span> </code></pre> <p>In addition to that its advisable to also use a type annotation. This is not a must, as the Elm compiler uses type inference to determine the types based on the argument usage in the program. Nevertheless its better to state the expected argument types explicitly as one ensures the correct type will be used and it also documents the program.</p> <p>Type annotations are prepended to the function definition and state the function name as well as the types of the arguments and of the return value.</p> <pre><code class="language-elm"><span class="hljs-title">sayHello</span> : <span class="hljs-type">String</span> -&gt; <span class="hljs-type">String</span> <span class="hljs-title">sayHello</span> name = <span class="hljs-type">String</span>.append <span class="hljs-string">"Hello "</span> name </code></pre> <p>To learn more about this specific feature, see the dedicated article about <a href="elm-type-annotations.html">type annotations in Elm</a>.</p> <h3>Anonymous functions / lambdas</h3> <p>Anonymous functions do not have a function definition. They are defined inline and are often used as arguments for functions like <code>List.map</code>. They are parenthesized and the content is prefixed with a backslash:</p> <pre><code class="language-elm">(\x y -&gt; x * y) <span class="hljs-comment">-- &lt;function&gt; : number -&gt; number -&gt; number</span> (\x y -&gt; x * y) <span class="hljs-number">2</span> <span class="hljs-number">3</span> <span class="hljs-comment">-- 6 : number</span> <span class="hljs-type">List</span>.map (\n -&gt; sayHello n) [<span class="hljs-string">"Alice"</span>, <span class="hljs-string">"Bob"</span>] <span class="hljs-comment">-- ["Hello Alice","Hello Bob"] : List String</span> </code></pre> <h3>Currying</h3> <p>Functions in Elm support <em>currying</em> – a technique of translating the evaluation of a function that takes multiple arguments into a function that takes a single argument.</p> <p>In the following example the function <code>threeTimes</code> encapsulates a partial call of <code>String.repeat</code> and returns a function that expects the last argument:</p> <pre><code class="language-elm"><span class="hljs-title">threeTimes</span> = <span class="hljs-type">String</span>.repeat <span class="hljs-number">3</span> <span class="hljs-comment">-- &lt;function&gt; : String -&gt; String</span> <span class="hljs-title">threeTimes</span> <span class="hljs-string">"hi"</span> <span class="hljs-comment">-- "hihihi" : String</span> </code></pre> <p>Currying is often used to prepare a function that expects multiple arguments for use as an argument of a function that passes only one argument (like <code>List.map</code>).</p> <p>For more examples and an in-depth explanation head over to LambdaCat: <a href="http://www.lambdacat.com/road-to-elm-currying-the-unknown/">Currying, The Unknown</a>.</p> <p>Now that you know about functions, let us have a closer look at <a href="/articles/elm-data-structures-list-array-set-dict.html">data structures in Elm</a>.</p>
